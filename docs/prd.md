# 当前阶段 PRD

## 1. 页面列表

1. 登录页：`/login`
2. Dashboard（所有任务）：`/tasks`
3. 新建任务：`/tasks/new`
4. 工作台：`/workspace/:task_id`

## 2. 页面目标与核心模块

## 2.1 登录页（/login）
目标：提供最简企业登录入口，登录后进入 Dashboard。

核心模块：

1. 公司代码输入
2. 账号输入
3. 密码输入
4. 登录按钮
5. 登录错误提示（轻量）

说明：

1. 不提供注册、找回密码、多因子认证。
2. 成功后跳转 `/tasks`。

## 2.2 Dashboard / 所有任务（/tasks）
目标：查看历史任务并进入任务处理。

核心模块：

1. 左侧导航（所有任务、新建任务）
2. 任务列表
3. 行内状态展示
4. 行点击进入工作台

任务列表字段（最小集）：

1. 任务名称
2. 导入表格名称
3. 导出模板名称
4. 创建时间
5. 状态（草稿/待处理/处理中/已完成/失败）
6. 最近更新时间

## 2.3 新建任务（/tasks/new）
目标：作为任务入口确认页，固定创建流程。

核心模块：

1. 任务名称
2. 导入文件上传
3. 导入模板类型（可选）
4. 导出模板选择（固定选项）
5. 备注（可选）
6. 创建任务按钮

创建结果：

1. 成功后直接跳转 `/workspace/:task_id`。
2. 失败时保留输入并提示。

## 2.4 工作台（/workspace/:task_id）
目标：承接任务后续处理流程，先做可落地骨架。

核心模块：

1. 顶部任务信息区
2. 导入数据概览区
3. 模板信息区
4. 处理结果/生成结果区
5. 执行状态/日志区（轻量）
6. 报告列表与下载入口
7. 失败提示/校验提示区

## 3. 页面跳转关系

1. `/login` 登录成功 -> `/tasks`
2. `/tasks` 点击“新建任务” -> `/tasks/new`
3. `/tasks` 点击任务行 -> `/workspace/:task_id`
4. `/tasks/new` 创建成功 -> `/workspace/:task_id`
5. `/workspace/:task_id` 返回任务列表 -> `/tasks`

## 4. 用户主流程

1. 用户登录
2. 进入 Dashboard 查看历史任务
3. 点击“新建任务”
4. 上传导入表格并选择导出模板
5. 点击创建任务
6. 进入工作台继续处理与查看结果

## 5. 关键字段说明

## 5.1 登录字段

1. `company_code`：公司代码
2. `username`：账号
3. `password`：密码

## 5.2 任务字段（展示）

1. `task_name`：任务名称
2. `import_file_name`：导入表格名
3. `export_template_name`：导出模板名
4. `status`：任务状态
5. `created_at`：创建时间
6. `updated_at`：最近更新时间

## 5.3 新建任务字段（提交）

1. `task_name`（必填）
2. `import_file_id`（必填）
3. `import_template_type`（可选）
4. `export_template_id`（必填）
5. `remark`（可选）

## 5.4 工作台字段（摘要）

1. `task_id`
2. `task_name`
3. `status`
4. `import_file_name`
5. `export_template_name`
6. `created_at`
7. `reports[]`（生成报告列表）

## 6. 当前阶段 验收标准

1. 可完成登录并进入 Dashboard。
2. Dashboard 可展示任务列表并进入工作台。
3. 新建任务页面可完成最小创建流程。
4. 创建后可进入对应任务工作台。
5. 工作台具备任务信息、结果区、日志区骨架。
