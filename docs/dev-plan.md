# 当前阶段开发计划

## 1. 当前现状基线（基于仓库）

已存在：

1. 后端 FastAPI 基础服务与 `/` 单入口页面（`backend/app/main.py`）
2. 当前前端为单页静态工具（`backend/app/static/index.html` + `js/app/*`）
3. 与模板/生成相关的后端服务和规则

需补充：

1. 登录页、Dashboard、新建任务、工作台的页面壳层与路由
2. 任务列表与新建任务闭环
3. 工作台骨架页
4. 当前阶段 文档

可先 mock：

1. 登录校验接口
2. 任务列表接口
3. 创建任务接口
4. 工作台摘要接口

## 2. 开发顺序（先主流程，后细节）

## 第 1 步：结构梳理与复用点确认

任务：

1. 确认现有静态资源组织与 API 客户端复用方式
2. 确认是否沿用现有 `backend/app/static` 作为前端落地点
3. 明确不重写现有生成业务逻辑

完成标准：

1. 输出“已存在/需补充/mock”清单
2. 确认 当前阶段 不影响现有核心生成链路

## 第 2 步：搭建基础路由与主框架

任务：

1. 增加登录页路由与页面
2. 增加主框架布局（左侧导航仅两项）
3. 增加 Dashboard 页面
4. 增加新建任务页面
5. 增加工作台占位页

依赖：

1. 无后端强依赖，可先前端静态与 mock 数据

完成标准：

1. 可在 4 个页面间按主流程跳转
2. 左侧导航仅“所有任务/新建任务”

## 第 3 步：打通任务列表与创建任务最小闭环

任务：

1. Dashboard 接任务列表（可 mock）
2. 新建任务表单提交（可 mock）
3. 创建成功后跳转工作台
4. 任务行点击进入工作台

依赖：

1. 需要统一任务状态枚举与任务 DTO

完成标准：

1. 完成“登录 -> 列表 -> 新建 -> 工作台”闭环
2. 列表字段完整展示

## 第 4 步：补齐工作台骨架

任务：

1. 顶部任务信息区
2. 导入数据概览区
3. 模板信息区
4. 处理结果区
5. 执行日志区（轻量）
6. 报告列表与下载入口占位

依赖：

1. 任务详情接口（可先 mock）

完成标准：

1. 工作台不是空页
2. 具备可承接后续业务扩展的稳定区块

## 第 5 步：文档、自检、收口

任务：

1. 完成 当前阶段 文档落地
2. 对照边界做越界检查
3. 统一命名与状态枚举

完成标准：

1. 文档可直接给合伙人/开发评审
2. 页面、流程、数据模型一致

## 3. 任务拆分（前端实施视角）

1. 路由层：登录守卫、页面路由、默认跳转
2. 布局层：主框架 + 左侧导航
3. 页面层：Login、Tasks、TaskCreate、Workspace
4. 组件层（最小拆分）：
   - `TaskListTable`
   - `TaskCreateForm`
   - `WorkspaceHeader`
   - `WorkspaceResultList`
   - `WorkspaceLogPanel`
5. 数据层：`authApi`、`taskApi`（先 mock，后替换真实接口）

## 4. 静态实现与 API 接入边界

先静态实现：

1. 页面结构与跳转
2. 表单校验与状态展示
3. 工作台骨架区块

后续接 API：

1. 登录鉴权
2. 任务列表拉取
3. 创建任务提交
4. 工作台详情与报告列表

## 5. 里程碑与优先级

1. P0：主流程闭环（登录 -> 列表 -> 新建 -> 工作台）
2. P1：工作台信息完整度（摘要/日志/结果区）
3. P2：接口替换与联调稳定性
