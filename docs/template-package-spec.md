# 输入/输出资料包接入规范（V1）

更新时间：2026-04-15

## 1. 目标
- 统一资料包接入方式，保证新包可直接进入工作台解析。
- 明确输入（识别）与输出（导出）的边界与规则。
- 明确多选记录时导出数据源范围。

## 2. 字段类型基线（统一）
全项目统一使用 5 类：

1. `date`
2. `string`
3. `number`
4. `check`
5. `signature`（jpg）

## 3. 输入资料包规范（input）
目录建议：

```text
backend/template-bundles/input/<bundle-id>/
  manifest.json
  schema.csv
  rules.json
```

必需文件：
- `manifest.json`：包元信息、入口文件声明
- `schema.csv`：字段分组与字段顺序
- `rules.json`：字段规则（含类型）

最小约束：
- `schema.csv` 第 1 行：分组名（可重复）
- `schema.csv` 第 2 行：字段名（唯一语义）
- `rules.json.field_rules` 覆盖 `schema.csv` 的第 2 行字段
- 字段类型只允许使用本规范 5 类（date/string/number/check/signature）

## 4. 输出资料包规范（output）
目录建议：

```text
backend/template-bundles/output/<bundle-id>/
  manifest.json
  template.docx
  rules.json   # V1建议新增，定义占位与填充规则
```

必需文件：
- `manifest.json`：输出包元信息、模板入口
- `template.docx`：导出模板

建议文件：
- `rules.json`：占位符 -> 字段来源映射、聚合规则、附表规则

## 5. 数据源范围规则（多选导出）
当工作台选中多条记录时：

- 导出数据源 `仅来自选中记录`
- 不得读取未选中记录作为补充数据源
- 右侧“模板字段编辑”的聚合值范围 = 选中记录集合
- “附表明细”逐行来源 = 选中记录集合（按当前排序或指定排序）

## 6. 钢瓶模板（steel-cylinder-v1）字段类型分层（V1）
说明：以下为业务分层基线，用于识别与展示对齐。

- `date`：
  - 检验日期
  - 上次检验日期
  - 下次检验日期
  - 制造年月（按日期语义管理）

- `string`：
  - 产权代码编号
  - 充装介质
  - 制造单位代码
  - 出厂编号
  - 检验员
  - 审核员
  - 备注

- `number`（带单位或数值量）：
  - 水压试验压力MPa
  - 公称工作压力MPa
  - 公称瓶重kg
  - 公称容积L
  - 设计壁厚mm
  - 实际重量kg
  - 重量损失率%
  - 实际容积L
  - 容积增大率%
  - 试验压力MPa（水压/气密）
  - 保压时间min（水压/气密）
  - 残余变形值mL
  - 容积全变形值mL
  - 残余变形率%

- `check`（打勾/结论/二选一）：
  - 余气处理
  - 外观清理检查
  - 音响检查
  - 内表面检查
  - 瓶口螺纹检查
  - 内部干燥
  - 试验结论（按检查结论语义）
  - 瓶阀检验（校阀/换阀）
  - 评定结论

- `signature`（jpg）：
  - 检验员签字图片
  - 审核签字图片
  - 批准签字图片

## 7. 接入验收清单（每个新资料包必跑）
1. 资料包结构完整（manifest/schema/rules 或 manifest/template）
2. 工作台能加载字段结构与分组
3. 字段类型可映射到 5 类
4. 单条与多条选择都可导出
5. 多选导出仅使用选中记录
6. 附表明细来自选中记录
7. signature 字段可注入 jpg 并导出

## 8. OCR 与字典协同策略（留档）
决策日期：2026-04-15

- OCR 必须持续启用，不允许被字典库替代。
- 字典库仅用于固定字段辅助归一，不用于覆盖全部识别流程。
- 当前固定字段（V1）：
  - 充装介质
  - 制造单位代码
  - 产权代码编号
  - 出厂编号

处理顺序：
1. 先执行 OCR，拿到原始识别值
2. 再执行字典归一（仅针对上述四字段）
3. 命中字典：写入标准值
4. 未命中字典：默认保留 OCR 原值（非严格模式）
5. 严格模式：未命中字典可置空

说明：
- 字典库文件：`backend/app/rules/steel_cylinder_value_library.json`
- 字典库是可维护“案底”，支持后续持续扩充。
