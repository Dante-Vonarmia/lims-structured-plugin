# 当前阶段项目边界

## 1. 阶段目标
在不引入复杂系统的前提下，打通最小可用闭环：

1. 用户可登录。
2. 登录后进入 Dashboard（所有任务）。
3. 可查看历史任务列表。
4. 可进入新建任务页面。
5. 可完成“导入表格 + 选择导出模板”并创建任务。
6. 创建后进入工作台。
7. 工作台可承接任务的后续处理与结果查看（骨架级）。

## 2. 本阶段要做什么

### 2.1 页面范围
仅包含以下页面：

1. 登录页（Login）
2. Dashboard / 所有任务页
3. 新建任务页
4. 工作台页

### 2.2 导航范围
左侧导航只保留两个入口：

1. 所有任务
2. 新建任务

### 2.3 流程范围
主流程固定为：

1. 登录
2. 进入 Dashboard
3. 查看任务
4. 点击新建任务
5. 上传导入表格 + 选择导出模板 + 创建
6. 进入工作台处理任务

### 2.4 登录方案（简化）
采用“公司代码 + 账号 + 密码”的简化企业凭证模式：

1. 登录字段：`company_code`、`username`、`password`
2. 校验方式：后端预置公司与账号（当前阶段 可先 mock）
3. 登录成功后写入会话（token/cookie 均可）并跳转 Dashboard

选择理由：

1. 能体现企业场景（非个人注册系统）。
2. 比纯用户名密码多一层组织归属约束，但不引入租户管理复杂度。
3. 与后续升级为正式企业认证兼容。

## 3. 本阶段不做什么
明确排除以下内容：

1. 复杂 RBAC 权限系统
2. 多角色后台管理
3. 模板在线编辑器
4. 企业组织架构管理
5. 消息中心
6. 设置中心
7. 审计系统
8. 复杂筛选报表系统
9. 与主流程无关的新页面/新菜单

## 4. 技术边界

1. 前端优先完成页面骨架与主流程交互，不做复杂状态编排。
2. API 可先提供 mock 契约，后续替换真实接口。
3. 模板选项来源固定模板库，不支持用户自定义模板设计。
4. 工作台只做骨架与必要区域，不实现复杂引擎、编排器、规则中心。

## 5. 业务边界声明
本系统在 当前阶段 不是通用任务系统，而是“模板匹配 + 数据转换 + 批量生成”场景入口壳层。

固定事实：

1. 输入来源是导入表格。
2. 输出是固定报告模板。
3. 一个任务可生成多个报告。
4. 报告由正文与附表组成。

## 6. 后续扩展位（仅预留，不实现）

1. 认证扩展：接入企业 SSO/OIDC、MFA、密码策略。
2. 权限扩展：按组织/角色做任务可见范围控制。
3. 模板扩展：模板版本管理、模板审核流程。
4. 工作台扩展：执行编排、校验规则中心、批处理队列、日志检索。
5. 结果扩展：报告批量下载、失败重试、结果归档。
