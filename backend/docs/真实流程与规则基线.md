# LIMS 插件真实流程与规则基线

更新时间：2026-03-27  
适用范围：当前 `backend` 代码与现有模板/原始记录/Excel 批量流程

## 1. 目标与边界
- 目标：先保证“现有原始记录 + 现有证书模板”都能跑通基础数据生成。
- 目标：不引入数据库，先用本地文件索引管理规则与资产。
- 边界：优先保证可生成与批量可跑通，不在本阶段做大规模模板精细字段定制。

## 2. 真实业务流程（当前实现）
1. 导入来源文件（`docx/xlsx/...`）。
2. 解析来源字段（器具名称、型号、编号、厂商等基础项）。
3. 匹配目标模板（模板列/模板代号/名称匹配/兜底空白模板）。
4. 允许编辑关键字段（部分模板有专用 editor schema）。
5. 生成目标 Word（优先走固定映射处理器，其次走通用记录填充）。
6. 下载或批量打包下载结果。

## 3. 基础字段（当前基线）
基础字段定义（本阶段“必须优先保证”）：
- `device_name`
- `manufacturer`
- `device_model`
- `device_code`
- `certificate_no`
- `basis_standard`
- `location`
- `temperature`
- `humidity`

说明：
- 上述字段已作为本地文档库索引字段。
- 对于模板无法精确命中的记录，允许先落到 `R-802B 空白.docx`，保证流程可跑通。

## 4. 模板匹配规则（当前优先级）
Excel 行级模板解析优先级：
1. 明确模板列（`template_name/模板/模板名称`）
2. 模板代号（`source_code/模板代号`）匹配
3. 基于行文本 + 器具名称做模板名匹配（`match_template_name`）
4. 调用方传入的默认模板
5. 兜底 `R-802B 空白.docx`

来源文本匹配补充：
- 已有模板库别名与关键词规则（`template_mapping_library.yaml`）。
- 已支持同义词归一（例如软击穿 -> 软化击穿）。

## 5. 文档抽取规则（当前基线）
已支持：
- 标签别名扩展：器具/设备/仪器名称、型号/规格、器具/设备编号、制造商等。
- 组合字段拆分：`型号/编号`、`Model/Number` 可拆为型号 + 编号。
- 跨行取值：标签右侧空值时，向后行邻近列补取。
- 标签噪声过滤：`Manufacturer`、`Instrument serial number` 等表头文本不作为真实值。
- 无效值兜底：必要时回退到“主要计量标准器具”首行信息。

## 6. 本地文档库（无数据库方案）
设计原则：
- 不建数据库，使用 JSON 文件管理“模板与原始记录基础结构索引”。

索引文件：
- `backend/outputs/local_document_library.json`

索引内容：
- 模板清单与基础标记命中情况（是否包含器具名/型号/编号等）
- 原始记录清单与基础字段覆盖度（coverage）
- 汇总统计

接口：
- `GET /api/library`：读取索引（可 `force_rebuild=true`）
- `POST /api/library/rebuild`：重建索引

## 7. XLSX 批量规则（当前基线）
已实现规则：
- 未指定 `sheet_name` 时，遍历全部 sheet（不是只读 active sheet）。
- 每行都构建 `raw_record`，用于模板自动匹配。
- 没有模板列时按第 4 节优先级自动分配模板。

批量可跑通定义：
- `parse_excel_rows` 无错误；
- 行可进入 `render_report` 并成功输出文档；
- 对未知器具允许落空白模板，避免流程中断。
