# Win11 客户交付方法（绿色版）

## 目标
- 客户端不安装 Python。
- 客户端不安装 Docker。
- 不走安装器（不写注册表安装项）。
- 客户只需要双击脚本即可使用。

## 交付物
交付整个目录：`release-win11`

目录内关键文件：
- `run_lims_backend.bat`：启动服务并自动打开浏览器。
- `stop_lims_backend.bat`：停止服务。
- `lims-backend/`：运行时目录（包含 Python 运行时和依赖）。

## 打包（在你自己的 Win11 机器）
1. 打开 PowerShell，进入目录：`backend/windows_bundle`
2. 执行：

```powershell
.\build_win11_bundle.ps1
```

3. 生成目录：`backend/release-win11`

## 客户使用（最终操作）
1. 解压你给他的 `release-win11` 文件夹。
2. 双击 `run_lims_backend.bat`。
3. 浏览器自动打开：`http://127.0.0.1:18081/`
4. 用完后双击 `stop_lims_backend.bat`。

## 说明
- 客户不需要手动双击 `.exe`。
- 后端仅监听本机地址 `127.0.0.1:18081`。
- 业务数据默认在：`release-win11\lims-backend\data\`

## 可能被企业策略拦截的点
- 如果公司策略禁止 `.bat` 或 PowerShell，需让 IT 白名单放行该目录。
- 若浏览器未自动打开，可手动访问：`http://127.0.0.1:18081/`
